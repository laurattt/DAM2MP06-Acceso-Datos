
1. Per què creus que l'enunciat demana utilitzar la biblioteca ws i WebSockets en lloc d'una API REST tradicional per enviar els moviments del jugador?
    RTA/ Porque existe una diferencia en su metodología de comunicación, API REST Recibe peticiones más no envía, en cambio WS puede enviar peticiones y viceversa, 
    exactamente ayuda en la mejoría del flujo de comunicación cuando hay una gran cantidad de datos (mejorar rta)


2. Com has implementat la recepció dels missatges en format JSON? Què passa si el client envia un missatge que no té un format JSON vàlid?
    RTA/ 
    
    La interacción de mensajes siempre pasa por un JSON.stringify, asi que no debería de existir alguna incidencia durante su flujo, sin embargo, el mensaje del cliente
    siempre pasara por un trycatch donde parsea el mensaje a JSON 

    CLIENTE
    ws.send(JSON.stringify({
        direction: inputManager.parseToDirection(),
        timestamp: Date.now()
    }));

    SERVER
    try {
        message = JSON.parse(data);
    } catch (err) {
        logger.error('JSON inválido'); --> En el caso de que falle se guardará en el fichero de los logs
        return;
    }
