
- LAZY: Carga de datos solo cuando son necesarios (muchos datos)
- EAGER: Carga de TODOS los datos de manera inmediata (puede generar sobrecarga, etc)

----------------------------------------------------------------------------

- OneToMany: Many SIEMPRE lleva la relación, ya que este consta con FK 
- ManyToMany: Tabla intermedia, pero

----------------------------------------------------------------------------

1. BIBLIOTECA:
    - OneToMany (Exemplars): 
        Por cada biblioteca hay muchos ejemplares 
        One lleva mapeo -> biblioteca + mappedBy

2. LIBRO:
    - ManyToMany(Autors)
        La relación la controla LIBRO -> IMPORTANTE
        JoinTable EXPLICITO por parte de libro 
        TABLA INTERMEDIA -> llibre_autor donde 
                            - JoinColumns lo maneja libro (llibre_id)
                            - InverseJoinColumns clase que usó mappedBy (autor_id)

    - OneToMany(Exemplars) 
        Por cada libro hay muchos ejemplares 
        Libro + mappedBy
        Exemplars + JoinColumn(llibre_id)

3. AUTOR:
    - ManyToMany(Llibres) 
    Libro MANEJA RELACIÓN
    Autor + mappedBy
    Libro + JoinTable(autor_id) -> inverse
        
4. EJEMPLARES:
    - ManyToOne(Llibre)
        JoinColumn(llibre_id)

    - ManyToOne(Biblioteca)
        JoinColumn(biblioteca_id)

    - OneToMany(Prestec)
        Ejemplares + mappedBy
        Prestec + JoinColumn(ejemplar_id)

5. PERSONA:
    - OneToMany(Prestamo)
    Por cada persona hay muchos prestamos
    Persona + mappedBy
    Prestamo JoinColumn(persona_id)

6. PRESTAMO:
    - ManyToOne(Exemplars)
        JoinColumn(ejemplar_id)

    - ManyToOne(Persona)
        JoinColumn(persona_id)


mappedBy: Nos indica que es el lado NO propietario de la relación
JoinTable: Controla relación
